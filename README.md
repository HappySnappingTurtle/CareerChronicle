<h1 align="center">Career Chronicle / 职业履历</h1>

<p align="center">
  一个围绕“职业 Build 构筑”的 Minecraft Forge RPG Mod。
</p>

<p align="center">
  <strong>不是选一条固定职业路线，而是把你的种族、职业履历、技能装配、装备和资源循环组合成一套可玩的 Build。</strong>
</p>

## 项目简介

Career Chronicle 是一个面向 Minecraft Forge 1.20.1 的 RPG 成长 Mod。它的核心不是“给玩家一个职业”，而是让玩家在长期游玩中积累多段职业履历，再把这些履历组合成不同的战斗构筑。

玩家进入世界后选择种族，之后通过战斗、探索、治疗、格挡、远程命中等行为获得职业经验。随着职业等级提升，玩家会在不同阶段选择职业段。每一段职业都会进入“履历池”，系统根据职业次数、标签强度和隐藏线索解锁技能、融合技能与隐藏职业。

真正进入战斗的不是完整技能池，而是玩家当前装配的技能槽、手中的职业武器、属性分配、法力/体力资源和冷却循环。也就是说，一个角色的强弱不只来自“选过什么职业”，更来自“你如何把这些职业能力装配成当前 Build”。

## 核心概念

```text
职业 Build = 种族起源 + 职业履历池 + 属性分配 + 技能装配 + 装备标签 + 资源循环
```

### 履历池

玩家最多形成 5 段职业履历。系统会把职业履历统计成：

- `class_counts`：每个职业被选择了几次
- `tag_scores`：火焰、近战、防御、弓术、神圣、暗影等标签强度
- `hidden_flags`：已经揭示的隐藏职业线索

同一组职业组合会得到同一组核心解锁，顺序只影响成型过程，不改变最终 Build 的核心身份。例如 `战士 + 火魔法师` 与 `火魔法师 + 战士` 都能走向同一种火焰近战共鸣。

### 技能装配

玩家可以解锁很多技能，但战斗中只能携带有限技能：

- 4 个主动技能槽
- 1 个终极槽
- 1 个种族槽
- 被动槽位已预留，后续扩展

技能池扩大的是构筑宽度，不是无条件堆高瞬间强度。玩家需要根据场景选择输出、控制、治疗、防御、位移或远程压制技能。

### 装备与资源

技能释放会检查装备、资源和冷却：

- 法术类技能需要法术焦点
- 远程技能需要远程焦点
- 近战技能需要近战武器
- 神圣技能需要神圣焦点
- 技能会消耗法力或体力
- 服务端会重新校验解锁、装配、冷却、装备和资源

这让职业武器不只是数值装备，而是 Build 的一部分。

## 种族

| 种族 | 构筑倾向 |
| --- | --- |
| 人类 | 适合快速尝试多职业融合 |
| 精灵 | 探索、远程、稳定视野 |

## 职业

当前职业包括：

- 战士
- 弓箭手
- 火魔法师
- 牧师
- 守护者
- 盗贼
- 冰魔法师
- 死灵法师
- 灰烬守望者
- 死亡骑士
- 巫妖

其中灰烬守望者、死亡骑士、巫妖属于隐藏职业，需要通过履历组合与隐藏线索揭示。

## 融合示例

| 职业组合 | 解锁方向 |
| --- | --- |
| 战士 + 火魔法师 | 炽焰冲锋 |
| 弓箭手 + 牧师 | 引光箭 |
| 冰魔法师 + 死灵法师 | 冻魂 |
| 盗贼 + 死灵法师 | 灵魂收割 |
| 守护者 + 牧师 | 神圣护盾 |
| 火魔法师 + 弓箭手 | 火焰箭 |

融合不是简单把两个职业名拼在一起，而是让两段职业经历在履历池中产生“共鸣”。

## 游戏内功能

### 界面与 HUD

- 首次进入世界时的种族选择界面
- 职业履历界面
- 职业预览、职业标签、技能预览
- 职业手册分页：概览、种族、职业、技能、线索、融合
- HUD 职业经验条
- 主动技能槽、冷却覆盖、法力/体力条
- 技能类型外框：主动、融合、隐藏、被动、终极、种族

### 默认键位

| 按键 | 功能 |
| --- | --- |
| `C` | 打开职业履历界面 |
| `Z` | 释放主动槽 1 |
| `X` | 释放主动槽 2 |
| `V` | 释放主动槽 3 |
| `B` | 释放主动槽 4 |

## 安装与运行

### 玩家安装

当前项目仍处于 Alpha 开发阶段。如果你已经有构建好的 jar：

1. 安装 Minecraft 1.20.1。
2. 安装 Forge 47.x。
3. 将 Mod jar 放入 `.minecraft/mods/`。
4. 启动游戏。

### 从源码构建

环境要求：

- JDK 17
- Minecraft 1.20.1
- Forge 47.4.20
- 使用仓库自带 Gradle Wrapper

常用命令：

```bash
./gradlew runClient
```

```bash
./gradlew build
```

```bash
./gradlew check
```

构建产物位于：

```text
build/libs/
```

## 开发者说明

### 数据目录

核心数据位于：

```text
src/main/resources/data/careerchronicle/careerchronicle/
```

| 目录 | 用途 |
| --- | --- |
| `races/` | 种族、特性、可选职业 |
| `classes/` | 职业、标签、基础技能、重复奖励、属性门槛 |
| `skills/` | 技能类型、资源消耗、冷却、装备需求、效果 |
| `fusions/` | 职业组合或标签条件到融合技能的映射 |
| `hidden_unlocks/` | 隐藏职业线索条件 |
| `xp_sources/` | 职业经验来源 |

客户端资源位于：

```text
src/main/resources/assets/careerchronicle/
```

包括语言文件、职业/种族/技能图标、物品贴图、投射物贴图、音效和物品模型。

### 代码结构

```text
src/main/java/com/hongyuwu/careerchronicle/
  career/      职业等级、职业段、履历池、融合与经验逻辑
  client/      职业界面、手册、HUD、技能图标和客户端表现
  command/     /career 调试与测试命令
```

## 世界观

Career Chronicle 的叙事隐喻是“编年史”：

| 机制 | 世界观词 |
| --- | --- |
| 职业段 | 篇章 |
| 职业履历池 | 编年录 |
| 技能 | 铭文 |
| 技能等级 | 铭文深度 |
| 融合技能 | 共鸣 |
| 隐藏职业 | 禁忌篇章 |
| 种族 | 第一篇章 |
| 装配槽 | 铭文石板 |
| 职业经验 | 编年之力 |

玩家不是在填写一张职业表，而是在把角色经历写成一部可以战斗的编年史。
