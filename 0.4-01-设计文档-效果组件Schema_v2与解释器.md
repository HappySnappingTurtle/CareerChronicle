## 一、架构设计

### 1.1 目标

将技能效果从硬编码（`SkillExecutorRegistry` 1864 行，73 个 if-else 方法）迁移为数据驱动的**效果组件序列**。技能 JSON 声明 `effects` 数组，Java 提供组件解释器。新增技能/调数值零 Java 改动。

### 1.3 核心接口

```java
// EffectOp.java
public interface EffectOp {
    /**
     * @param ctx    执行上下文（caster、targets、skill level、trigger 阶段）
     * @param params 该 op 在技能 JSON effects 数组中的参数对象
     */
    void apply(EffectContext ctx, JsonObject params);
}

// EffectContext.java
public record EffectContext(
    ServerPlayer caster,
    List<LivingEntity> targets,    // 对 aoe/projectile_hit 有值，对 cast 为空
    int skillLevel,                // 方案A派生值，由 SkillLevelService 计算
    String trigger,                // "cast" | "projectile_hit" | "tick"
    ResourceLocation skillId,
    ServerLevel level
) {
    /** 创建仅含 caster 的初始上下文 */
    public static EffectContext castContext(ServerPlayer caster, int skillLevel, ResourceLocation skillId) {
        return new EffectContext(
            caster, List.of(), skillLevel, "cast",
            skillId, (ServerLevel) caster.level()
        );
    }

    /** 用新 targets 和 trigger 派生子上下文 */
    public EffectContext withTargets(List<LivingEntity> targets, String trigger) {
        return new EffectContext(caster, targets, skillLevel, trigger, skillId, level);
    }
}
```

### 1.4 组件注册表

```java
// EffectOpRegistry.java
public final class EffectOpRegistry {
    private static final Map<String, EffectOp> OPS = new LinkedHashMap<>();

    public static void register(String name, EffectOp op) {
        if (OPS.containsKey(name)) {
            throw new IllegalStateException("Duplicate EffectOp: " + name);
        }
        OPS.put(name, op);
    }

    public static EffectOp get(String name) {
        return OPS.get(name);
    }

    public static boolean exists(String name) {
        return OPS.containsKey(name);
    }

    public static Set<String> allNames() {
        return Set.copyOf(OPS.keySet());
    }

    /** 在 mod 初始化时调用，注册内置 ops */
    public static void registerBuiltins() {
        register("damage", new DamageOp());
        register("apply_effect", new ApplyEffectOp());
        register("heal", new HealOp());
        register("knockback", new KnockbackOp());
        register("projectile", new ProjectileOp());
        register("aoe", new AoeOp());
        register("dash", new DashOp());
        register("shield", new ShieldOp());
        register("resource", new ResourceOp());
        register("summon", new SummonOp());   // stub
    }
}
```

注册时机：`CareerChronicleMod.commonSetup()` 中调用 `EffectOpRegistry.registerBuiltins()`，在 `NetworkHandler.register()` 之前。其他 MOD 可在自己的 `commonSetup` 中调用 `register()` 扩展。

### 1.5 组件序列执行器

```java
// EffectPipeline.java
public final class EffectPipeline {
    /**
     * 执行技能 JSON 中 effects 数组的所有组件。
     * 按数组顺序同步执行，遇错 log 并跳过该 op（不中断后续）。
     * "on" 字段做 trigger 过滤：当前 trigger != op.on 则跳过。
     * 返回 true 如果至少一个 op 成功执行。
     */
    public static boolean execute(EffectContext ctx, List<JsonObject> effects) {
        boolean any = false;
        for (JsonObject effect : effects) {
            String opName = effect.get("op").getAsString();
            String on = effect.has("on") ? effect.get("on").getAsString() : "cast";
            if (!on.equals(ctx.trigger())) {
                continue;
            }
            EffectOp op = EffectOpRegistry.get(opName);
            if (op == null) {
                CareerChronicleMod.LOGGER.warn("Unknown effect op '{}' in skill {}", opName, ctx.skillId());
                continue;
            }
            try {
                op.apply(ctx, effect);
                any = true;
            } catch (Exception e) {
                CareerChronicleMod.LOGGER.error("Error executing op '{}' in skill {}", opName, ctx.skillId(), e);
            }
        }
        return any;
    }
}
```

## 二、首批 10 个效果组件详细设计

### 2.1 damage

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| amount | float | 必填 | 基础伤害 |
| per_level | float | 0 | 每等级额外伤害 |
| damage_type | string | "player_attack" | 伤害源类型：`player_attack` / `magic` / `fire` |
| on | string | "cast" | 触发阶段 |

**实际伤害** = `amount + per_level * (skillLevel - 1)`

**逻辑**：对 ctx.targets 中每个存活目标调用 `target.hurt(damageSource, actualAmount)`。damageSource 由 damage_type 映射：player_attack→`caster.damageSources().playerAttack(caster)`，magic→`caster.damageSources().magic()`，fire→`caster.damageSources().inFire()`。

**注意**：触发阶段为 `cast` 时 targets 来自前置 `aoe` op 的选中集；为 `projectile_hit` 时 targets 由投射物命中事件传入。

### 2.2 apply_effect

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| effect | string | 必填 | 药水效果 id（如 `minecraft:slowness`） |
| duration | int | 必填 | 持续 tick |
| amplifier | int | 0 | 效果等级（0=I） |
| per_level_duration | int | 0 | 每等级额外 tick |
| target | string | "enemy" | `enemy`→ctx.targets / `self`→caster |
| on | string | "cast" | 触发阶段 |

**实际持续** = `duration + per_level_duration * (skillLevel - 1)`

### 2.3 heal

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| amount | float | 必填 | 基础治疗量 |
| per_level | float | 0 | 每等级额外 |
| target | string | "self" | `self` / `ally_area` |
| radius | double | 3.5 | `ally_area` 时的范围 |
| on | string | "cast" | 触发阶段 |

**逻辑**：`target.heal(actualAmount)`。`ally_area` 时对范围内 `isAlliedTo(caster) || target == caster` 的存活实体治疗。同时调用 `CareerProgressionService.awardBehaviorXp` 奖励治疗经验。

### 2.4 knockback

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| strength | double | 必填 | 击退强度 |
| direction | string | "away" | `away`（远离 caster）/ `look`（沿 caster 视线） |
| on | string | "cast" | 触发阶段 |

**逻辑**：对 ctx.targets 中每个目标调用 `target.knockback(strength, dx, dz)`。`away`：dx = caster.x - target.x, dz = caster.z - target.z。`look`：dx/dz 从 caster.getLookAngle()。

### 2.5 projectile

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| entity | string | 必填 | 投射物 entity id（`careerchronicle:career_fireball` 等） |
| speed | float | 1.5 | 初速 |
| count | int | 1 | 数量 |
| spread | float | 0 | 偏转角度（scatter 用） |
| damage | float | 6.0 | 传递给 CareerProjectileEntity 的伤害 |
| per_level_damage | float | 0 | 每等级额外伤害 |
| burn_seconds | int | 0 | 着火秒数 |
| on | string | "cast" | 必须为 "cast" |

**逻辑**：复用现有 `CareerProjectileEntity` 构造函数。`count > 1` 时用 `spread` 角度均匀分布。使用现有 `castOrigin` 和 `aimDirection` 辅助方法（从 SkillExecutorRegistry 提取为公共静态方法到一个新的 `SkillCastHelper` 工具类）。

投射物命中时由 `CareerProjectileEntity.onHit` 构造 `projectile_hit` trigger 的 EffectContext，再次调用 `EffectPipeline.execute` 处理 `on: "projectile_hit"` 的 ops。

### 2.6 aoe

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| radius | double | 3.0 | 半径 |
| per_level_radius | double | 0 | 每等级额外半径 |
| height | double | 1.0 | 垂直范围 |
| filter | string | "hostile" | `hostile` / `ally` / `all` |
| on | string | "cast" | 触发阶段 |

**逻辑**：不直接造成效果，而是**设置 ctx.targets**。用 `level.getEntitiesOfClass(LivingEntity.class, bbox.inflate(radius, height, radius), predicate)` 搜集目标，后续 ops 作用于此集合。predicate 由 filter 决定：`hostile`→`canAffectHostileTarget`（复用现有方法）；`ally`→`isAlliedTo || ==caster`；`all`→isAlive。

**设计关键**：aoe 与后续 damage/apply_effect 通过共享的 targets 列表协作。EffectPipeline 在遇到 aoe op 时将选中实体设入上下文，后续 op 自动作用于选中集。实现方式：EffectPipeline 维护一个可变 targets 列表，aoe op 通过回调设置它。

### 2.7 dash

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| distance | double | 必填 | 位移距离 |
| direction | string | "look" | `look`（沿视线）/ `backward`（后退） |
| vertical | double | 0.18 | 垂直分量 |
| on | string | "cast" | 触发阶段 |

**逻辑**：`caster.push(look * distance, vertical, look * distance)` + `caster.hurtMarked = true`。

### 2.8 shield

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| amount | float | 必填 | 吸收量 |
| per_level | float | 0 | 每等级额外 |
| duration | int | 200 | 持续 tick |
| on | string | "cast" | 触发阶段 |

**逻辑**：`caster.addEffect(new MobEffectInstance(MobEffects.ABSORPTION, duration, amplifierFromAmount))`。amplifier 计算：absorption 每级 +4 HP，`amplifier = floor(actualAmount / 4) - 1`，clamped to [0, 4]。

### 2.9 resource

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| type | string | 必填 | `mana` / `stamina` |
| amount | int | 必填 | 正数=回复，负数=消耗（在 cost 之外额外消耗） |
| per_level | int | 0 | 每等级额外 |
| on | string | "cast" | 触发阶段 |

### 2.10 summon（0.5+ 占位）

| 参数 | 类型 | 默认 | 说明 |
|---|---|---|---|
| entity | string | — | 召唤物 entity id |
| count | int | 1 | 数量 |
| lifetime_ticks | int | 600 | 存活时间 |
| cap | int | 3 | 同类上限 |

**当前实现**：注册但 `apply()` 为空（log warning "summon op not yet implemented"）。0.5 落地时实现。

## 四、SkillLevelService 设计（方案 A）

### 4.1 接口

```java
public final class SkillLevelService {
    /**
     * 运行时派生技能等级。纯函数，不存储。
     * level = min(maxLevel, 1 + countExcess)
     * countExcess = 数据源的当前值 - 技能解锁所需的最低值
     */
    public static int levelOf(ICareerData data, SkillDef skill) {
        if (skill.upgrade() == null || UpgradeRule.NONE.equals(skill.upgrade())) {
            return 1;
        }
        return levelOf(data, skill.upgrade());
    }

    public static int levelOf(ICareerData data, UpgradeRule rule) {
        if (rule == null || "none".equals(rule.source())) {
            return 1;
        }
        int score = resolveScore(data, rule.source());
        // 初始解锁 = score >= 1，之后每多 1 点 score 升 1 级
        int level = Math.max(1, score);
        return Math.min(rule.maxLevel(), level);
    }

    /** 解析 source 字符串获取分数 */
    private static int resolveScore(ICareerData data, String source) {
        // "tag:fire" → tagScores 中 fire 标签的值
        // "class:fire_mage" → classCounts 中 fire_mage 的值
        if (source.startsWith("tag:")) {
            ResourceLocation tagId = ResourceLocation.tryParse(
                CareerChronicleMod.MOD_ID + ":" + source.substring(4));
            return tagScoreFromHistory(data, tagId);
        }
        if (source.startsWith("class:")) {
            ResourceLocation classId = ResourceLocation.tryParse(
                CareerChronicleMod.MOD_ID + ":" + source.substring(6));
            return classCountFromHistory(data, classId);
        }
        return 1;
    }

    /** 从 classHistory 计算 tag 总分（与 CareerProgressionService.historyScores 相同逻辑） */
    private static int tagScoreFromHistory(ICareerData data, ResourceLocation tagId) {
        RegistrySnapshot registry = CareerRegistry.snapshot();
        int score = 0;
        for (ResourceLocation classId : data.getClassHistory()) {
            ClassDef classDef = registry.classes().get(classId);
            if (classDef != null && classDef.tags().contains(tagId)) {
                score++;
            }
        }
        return score;
    }

    private static int classCountFromHistory(ICareerData data, ResourceLocation classId) {
        int count = 0;
        for (ResourceLocation id : data.getClassHistory()) {
            if (id.equals(classId)) {
                count++;
            }
        }
        return count;
    }
}
```

### 4.2 无序性证明

`levelOf` 仅依赖 `data.getClassHistory()` 的**多重集统计**（classCounts / tagScores），与列表顺序无关。证明：对于任何两个排列 P1、P2，其 classCounts 和 tagScores 相同 ⇒ `resolveScore` 返回相同值 ⇒ `levelOf` 返回相同值。满足基线 B2。

### 4.3 客户端派生

客户端 `ClientCareerData.snapshot()` 中包含 `classHistory`，客户端可用同一纯函数计算显示用等级（UI 显示"火球术 Lv.3"），无需额外 S2C 同步字段。

## 五、数据流

```
玩家按技能键
  → C2SUseSkillPacket → 服务端
  → CareerSkillService.useKnownSkill()
    → 校验链（存活→解锁→装配→冷却→装备→资源）
    → if skill.effects() 非空:
        → SkillLevelService.levelOf(data, skill)    // 派生等级
        → EffectContext.castContext(player, level, skillId)
        → EffectPipeline.execute(ctx, skill.effects())
          → 逐 op 执行（damage/aoe/projectile/...）
          → aoe op 搜集 targets 注入上下文
          → projectile op 创建实体（命中时触发 projectile_hit 上下文的二次 pipeline）
        → playComponentFx(player, skill.fx())       // 音效/粒子/震动
      else:
        → SkillExecutorRegistry.execute()            // 旧路径
    → consumeResource / setCooldown / sync
```

## 六、CareerDataParsers 修改

`CareerDataParsers.skill()` 扩展为：

```java
static SkillDef skill(ResourceLocation id, JsonObject json) {
    List<JsonObject> effects = JsonDataUtil.objectList(json, "effects");
    UpgradeRule upgrade = parseUpgradeRule(json);
    FxSpec fx = parseFxSpec(json);

    return new SkillDef(
        id,
        JsonDataUtil.optionalString(json, "display_key", "skill." + id.getNamespace() + "." + id.getPath()),
        JsonDataUtil.optionalString(json, "type", "active"),
        JsonDataUtil.optionalString(json, "resource", "none"),
        JsonDataUtil.optionalInt(json, "resource_cost", 0),
        effects.isEmpty() ? JsonDataUtil.id(JsonDataUtil.string(json, "executor")) : null,
        JsonDataUtil.optionalInt(json, "cooldown_ticks", 0),
        skillRequirements(json),
        effects,
        upgrade,
        fx
    );
}

private static UpgradeRule parseUpgradeRule(JsonObject json) {
    if (!json.has("upgrade")) return UpgradeRule.NONE;
    JsonObject obj = json.getAsJsonObject("upgrade");
    return new UpgradeRule(
        JsonDataUtil.optionalString(obj, "source", "none"),
        JsonDataUtil.optionalInt(obj, "max_level", 1)
    );
}

private static FxSpec parseFxSpec(JsonObject json) {
    if (!json.has("fx")) return FxSpec.EMPTY;
    JsonObject obj = json.getAsJsonObject("fx");
    return new FxSpec(
        JsonDataUtil.optionalString(obj, "cast_sound", null),
        JsonDataUtil.optionalString(obj, "cast_particle", null),
        JsonDataUtil.optionalString(obj, "hit_sound", null),
        JsonDataUtil.optionalString(obj, "hit_particle", null),
        (float) JsonDataUtil.optionalDouble(obj, "camera_shake", 0),
        JsonDataUtil.optionalInt(obj, "camera_shake_ticks", 0),
        JsonDataUtil.optionalBoolean(obj, "cast_circle", false)
    );
}
```

**关键兼容点**：当 `effects` 存在时 `executor` 允许为 null——`SkillDef` compact constructor 中 `executor` 的非空校验需改为条件校验。

## 七、迁移策略

## 八、性能与安全

- **性能**：EffectPipeline 同步执行，无额外线程/调度开销。组件解析在 reload 时完成（JSON → List\<JsonObject\>），运行时无反射/字符串解析。
- **服务端权威（B4）**：所有 op.apply() 在服务端执行。客户端仅通过 S2C fx 包播放视听，不执行逻辑。projectile_hit 上下文由服务端 `CareerProjectileEntity.onHit` 触发——现有机制不变。
- **数据安全**：`CareerRegistryValidator` 在 reload 时校验所有 op 名已注册、effects 格式合法。失败则保留旧 registry（原子替换机制不变）。
